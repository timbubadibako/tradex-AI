## 2. Shared Endpoints (Routed directly from `routes.py`)

Because `dummy_main.py` uses `app.include_router(router)`, these endpoints behave exactly the same in both Simulation and Production modes.

### Telemetry & Logs
- **`GET /api/events`**: Returns `global_event_log` for the Evolution Feed.
- **`GET /api/trades`**: Returns RAM-based `trade_history` for the active session.
- **`GET /api/persistent_trades`**: Fetches the permanent Ledger from Supabase.
- **`GET /api/checkpoints`**: Fetches Neural Evolution milestones from Supabase.
- **`GET /api/safety`**: Returns `safety_settings` (ATR Multipliers).

### Control Tuas (POST Requests)
- **`POST /api/set_coin?coin=BTC`**: Changes the active colony focus.
- **`POST /api/set_timeframe?tf=1h`**: Switches between Macro/Micro analysis.
- **`POST /api/update_manual_config`**: Updates confidence/signal thresholds.
- **`POST /api/update_daily_target`**: Modifies the anchor for the Target Engine.
- **`POST /api/update_safety`**: Changes Stop-Loss and Take-Profit bounds.
- **`POST /api/panic_sell`**: Triggers emergency liquidation protocol.
- **`POST /api/manual_buy` / `POST /api/manual_sell`**: CFO tactical overrides.

## Conclusion
The API Contract is strictly enforced. The frontend UI does not need to know whether it is talking to `dummy_main.py` or `main.py`. Data mappings are 1:1, null-safe, and production-ready.
